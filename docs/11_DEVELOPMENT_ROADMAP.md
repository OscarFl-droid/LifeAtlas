# Development Roadmap

## Phase 0 – Canonical definition
Freeze language, boundaries, target demographic, and first domains.

## Phase 1 – Engine foundation
Event schema, local database, replay, sequence rules, states, thresholds, Atlas node model, deterministic tests.

## Phase 2 – Offline shell
Installable PWA, service worker, IndexedDB or SQLite, encrypted backup, migrations, import/export.

## Phase 3 – Evidence system
Evidence objects, rule registry, confidence, source display, review workflow, rule versioning.

## Phase 4 – First user experience
Onboarding, one-tap logging, daily view, Atlas, explanations, recommendations, corrections.

## Phase 5 – Three demonstration packs
Routine/recovery, professional development, study consistency.

## Phase 6 – Pilot validation
Usability, burden, retention, usefulness, trust, recommendation relevance, expert agreement, offline reliability.

## Phase 7 – Commercial pilot
One demographic, one primary use case, paid pilot, evidence board, privacy/legal review, pricing validation.
